Trackspace
Technical stack brief | Updated May 30, 2026

Recommended Tech Stack

Trackspace should be a source-backed lunar base progress tracker with a 3D command surface, dependency graph, timeline, and curated data model. Keep the stack portable and avoid user/auth complexity early.

Core Recommendation

Layer Choice Reason
App framework Next.js + TypeScript Routes, server handlers, static/docs pages, and API endpoints in one app.
UI styling Tailwind CSS + custom components Fast to build, easy to keep minimal, avoids heavy UI framework lock-in.
3D command scene Three.js + @react-three/fiber + @react-three/drei Best fit for custom Earth/Moon scene, trajectories, overlays, and timeline playback.
Dependency map @xyflow/react Purpose-built interactive node/edge graph for capability dependencies.
Database File-based SQLite (via better-sqlite3) Read-mostly curated MVP dataset needs zero infra. Spatial anchors are plain lat/lng numeric columns — no PostGIS until a real spatial query exists. Schema stays portable to Postgres later.
Database access Drizzle ORM Typed schema, SQL-friendly queries, migrations, and one schema that targets SQLite now and Postgres later with minimal change.
Server/client data TanStack Query Cache, loading, refetch, and error states for timeline/map/detail data.
Local UI state Zustand Simple state for selected object, filters, drawer state, and timeline position.
Validation Zod Validate ingested data, API payloads, and source-backed claim records.
Testing Vitest Unit tests for data logic, schema helpers, ingestion transforms, and impact calculations.

Use Now

These should be in the first implementation.

  • @xyflow/react
  • drizzle-orm, drizzle-kit, better-sqlite3

Peer-dep note: @react-three/fiber v9 declares a React ceiling of <19.3. It installs cleanly against the React 19.2.x that create-next-app ships today, but once the scaffold starts pulling React 19.3+, npm install will throw an ERESOLVE peer conflict until fiber raises the ceiling. Fix when it happens: npm install --legacy-peer-deps.

Defer

Useful later, but not needed for the MVP.

  • PostgreSQL (Dockerized or managed) once the dataset grows or ingestion lands.
  • PostGIS only when a feature needs real spatial queries (radius, polygon, intersection).
  • CesiumJS for high-fidelity geospatial visualization.
  • Auth, users, saved dashboards, and alerts.
  • Automated ingestion workers beyond seed/manual scripts.
  • Search infrastructure beyond the database's built-in full-text search (SQLite FTS5, later Postgres FTS).
  • Realtime updates or websockets.

Database Stance

Start on file-based SQLite. The MVP is a small, curated, read-mostly dataset, so there is no payoff to running a Postgres container during the build phase — SQLite gets the app running with zero infra and no "is Docker up?" friction. Spatial anchors are stored as plain lat/lng numeric columns and rendered directly in the 3D scene; none of the four screens needs real spatial SQL, so PostGIS is deferred until a feature actually requires it. Keep the Drizzle schema portable so it can move to any managed Postgres provider when the dataset grows, ingestion lands, or the app goes public. Supabase is not required: no user-specific features, no auth, no realtime.

Data Model

Table Purpose
milestones Artemis missions, lunar buildout phases, demos, reviews, and major program checkpoints.
capabilities Landing, launch, suits, mobility, power, comms, habitats, logistics, and supporting systems.
events Tests, launches, incidents, announcements, slips, reviews, deliveries, and demonstrations.
sources NASA, FAA, OIG, GAO, contractor updates, reputable reporting, and source metadata.
claims Atomic source-backed facts and inferences used to justify readiness and impact labels.
locations Launch pads, test stands, lunar regions, Gateway context, and other spatial anchors.
dependencies Edges between capabilities, milestones, events, and locations for the dependency map.

Implementation Phases

Status Phase Scope
Now Static MVP data SQLite file, Drizzle schema, seed data, four app screens, event drawer.
Now Interactive visuals 3D Earth/Moon scene, dependency map, timeline filtering, selected-object panel.
Later Ingestion scripts NASA feeds, FAA updates, Launch Library 2, contractor news, manual review queue.
Later Deployment Migrate the portable Drizzle schema from SQLite to managed Postgres when the app needs to be public.
Avoid Premature platform features No auth, no user accounts, no realtime, no alert system until the core tracker works.
This stack keeps Trackspace local-first, source-backed, and portable while leaving room for richer geospatial and ingestion features later.
