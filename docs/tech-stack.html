Trackspace
Technical stack brief | Updated May 30, 2026

Recommended Tech Stack

Trackspace should be a source-backed lunar base progress tracker with a 3D command surface, dependency graph, timeline, and curated data model. Keep the stack portable and avoid user/auth complexity early.

Core Recommendation

Layer Choice Reason
Package manager and runtime Bun One tool for dependency installation, project scripts, local Next.js processes, and Vercel Functions.
App framework Next.js + TypeScript Routes, server handlers, static/docs pages, and API endpoints in one app.
UI styling Tailwind CSS + custom components Fast to build, easy to keep minimal, avoids heavy UI framework lock-in.
3D command scene Three.js + @react-three/fiber + @react-three/drei Best fit for custom Earth/Moon scene, trajectories, overlays, and timeline playback.
Dependency map @xyflow/react Purpose-built interactive node/edge graph for capability dependencies.
Database Convex Realtime document store for the curated dataset, ingestion-run history, and discovery queue. Spatial anchors remain plain lat/lng numeric fields until a real spatial query exists.
Server/client data convex/react Live queries push ingestion updates into the dashboard without polling or cache invalidation code.
Local UI state Zustand Simple state for selected object, filters, drawer state, and timeline position.
Validation Zod Validate ingested data, API payloads, and source-backed claim records.
Testing Vitest Unit tests for data logic, schema helpers, ingestion transforms, and impact calculations.

Use Now

These should be in the first implementation.

  • next, react, typescript
  • tailwindcss
  • three, @react-three/fiber, @react-three/drei
  • @xyflow/react
  • convex
  • @tanstack/react-query, zustand, zod

Peer-dep note: @react-three/fiber v9 declares a React ceiling of <19.3. It installs cleanly against the React 19.2.x used today. If a future React release exceeds that ceiling, keep React pinned to the latest compatible version until Fiber expands its peer range, then run bun install again.

Defer

Useful later, but not needed for the MVP.

  • PostGIS only when a feature needs real spatial queries (radius, polygon, intersection).
  • CesiumJS for high-fidelity geospatial visualization.
  • Auth, users, saved dashboards, and alerts.
  • Search infrastructure beyond Convex indexes when full-text search becomes a real product need.

Database Stance

Use Convex as the backend of record. The dataset is small and source-backed, but the app benefits from live updates after ingestion runs and from scheduled backend jobs without a separate cron service. Spatial anchors stay as plain lat/lng numeric fields until a feature actually needs real spatial queries.

Data Model

Table Purpose
milestones Artemis missions, lunar buildout phases, demos, reviews, and major program checkpoints.
capabilities Landing, launch, suits, mobility, power, comms, habitats, logistics, and supporting systems.
events Tests, launches, incidents, announcements, slips, reviews, deliveries, and demonstrations.
sources NASA, FAA, OIG, GAO, contractor updates, reputable reporting, and source metadata.
claims Atomic source-backed facts and inferences used to justify readiness and impact labels.
locations Launch pads, test stands, lunar regions, Gateway context, and other spatial anchors.
dependencies Edges between capabilities, milestones, events, and locations for the dependency map.

Implementation Phases

Status Phase Scope
Now Static MVP data Convex schema, seed data, four app screens, event drawer.
Now Interactive visuals 3D Earth/Moon scene, dependency map, timeline filtering, selected-object panel.
Now Ingestion scripts NASA feeds, Launch Library 2, scheduled Convex refresh, manual review queue.
Later Deployment Link the Convex deployment to the production app and set matching ingest secrets.
Avoid Premature platform features No auth, no user accounts, no alert system until the core tracker works.
This stack keeps Trackspace local-first, source-backed, and portable while leaving room for richer geospatial and ingestion features later.
